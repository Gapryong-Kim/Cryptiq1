Cryptography Overview

Cryptography is the practice of disguising information so that only the intended reader can understand it. Every classical cipher is based on one of two main ideas:

  • Substitution ‚Äî swapping each letter for a different letter or symbol.
  • Transposition ‚Äî keeping the letters but changing their order.

Substitution Cipher

Learn more ‚Üí

A substitution cipher replaces every letter in the alphabet with another. The mapping stays the same for the entire message ‚Äî that consistency is exactly what you exploit to break it.

Plaintext: HELLO WORLD Ciphertext: QEBBY TBOBD

How to Break It (fast)

Use frequency + word patterns. In English, E/T/A/O/I are common. If spaces are kept, word lengths and repeats give huge clues.

What the frequency looks like: Monogram frequencies keep the same overall ‚ÄúEnglish shape‚Äù (one letter is still the most common, a few are medium, many are rare), but the labels are swapped. Bigrams/trigrams also look English-ish but with letters renamed.

Deep dive

What leaks (weakness)

  • Letter frequency: ciphertext inherits English distribution.
  • Repeats: double letters (LL, EE) and repeated short words.
  • Word shapes: patterns like _H_ can hint ‚ÄúTHE‚Äù.

Practical breaking steps

  • Run frequency: guess E/T/A/O/I candidates.
  • Find ‚ÄúTHE‚Äù, ‚ÄúAND‚Äù, ‚ÄúOF‚Äù, ‚ÄúTO‚Äù using patterns.
  • Lock confirmed letters, then iterate (it snowballs).

Caesar Cipher

Learn more ‚Üí

The Caesar cipher shifts every letter by the same number of positions in the alphabet. After Z, it wraps back to A.

Shift: 3 Plaintext: HELLO ‚Üí Ciphertext: KHOOR

How to Break It (fast)

There are only 25 possible shifts. Brute force them and pick the one that produces readable English.

What the frequency looks like: Exactly like normal English, just shifted: the frequency curve is unchanged. A chi‚Äësquare match against English over all 25 shifts usually pops the right one immediately.

Deep dive

How it works (math)

Convert letters to numbers A=0..Z=25. Encrypt: (x + s) mod 26. Decrypt: (x ‚àí s) mod 26.

Fast sanity checks

  • Common hits: ‚ÄúTHE‚Äù, ‚ÄúAND‚Äù, ‚ÄúING‚Äù.
  • ROT13 is special: shift 13 decrypts itself.

Learn more ‚Üí

Affine encrypts letters using multiplication and addition mod 26. It‚Äôs stronger than Caesar, but still small enough to brute force.

Playfair Cipher

Learn more ‚Üí

Playfair encrypts pairs of letters using a 5√ó5 key square (I/J combined). It hides single-letter frequencies ‚Äî but digraph patterns leak.

Key: MONARCHY HELLO ‚Üí GYIYQ

How to Break It (fast)

Use digraph frequency + cribs. For serious cracking, heuristic search (hill-climbing) is common.

What the frequency looks like: Single-letter frequencies are less helpful because encryption works on digraphs. You‚Äôll often see fewer obvious English bigrams, no doubled letters (like ‚ÄúLL‚Äù), and digraph patterns dominate instead.

Deep dive

Rules

  • Same row: take letters to the right (wrap).
  • Same column: take letters below (wrap).
  • Rectangle: swap corners (same row, other column).

Plaintext prep

  • Split into pairs.
  • If a pair repeats (LL), insert filler (often X): HE LX LO.

Breaking

  • Look at digraphs like TH/HE/IN/ER.
  • Try cribs and test quickly with tools.

Hill Cipher

Learn more ‚Üí

The Hill cipher encrypts blocks of letters using matrix multiplication mod 26. It‚Äôs a classical cipher that directly uses linear algebra.

Key Matrix: [[3,3],[2,5]] Plaintext (HI) ‚Üí Ciphertext (RC)

How to Break It (fast)

If you know matching plaintext/ciphertext blocks, you can solve for the matrix. Otherwise, brute force only works for tiny keys.

Deep dive

How it works

  • Choose block size n (e.g. 2).
  • Convert letters to numbers.
  • Compute C = K¬∑P mod 26.

Key requirement

K must be invertible mod 26 (det(K) must have an inverse mod 26).

Breaking

  • Known plaintext: enough pairs ‚Üí solve for K.
  • Otherwise: use automated search with strong scoring.

Transposition Ciphers

Learn more ‚Üí

Transposition ciphers rearrange letters instead of replacing them. Letter frequency stays roughly English, but words are scrambled.

Key: ZEBRAS Plaintext: WEAREDISCOVEREDRUN Ciphertext: EVLNEACDTKESEAQROFO

How to Break It (fast)

  • For columnar: test short keys or use heuristic search for longer keys.
  • Cribs: try placing a suspected word into a grid.

Bacon Cipher

Learn more ‚Üí

Bacon‚Äôs cipher hides letters as patterns of As and Bs ‚Äî often disguised using two text styles (case, font, bold/normal).

How to Break It (fast)

Identify the two styles, map them to A/B, group into 5s, and decode.

Deep dive

Decoding steps

  • Pick a rule: Style 1 = A, Style 2 = B.
  • Read off the A/B stream.
  • Group into chunks of 5.
  • Convert each chunk into a letter (variant-dependent).

Common gotcha

Copy/paste often destroys formatting. If styles disappear, use screenshots or inspect the HTML.

Frequency analysis

Frequency analysis is the classic first step in breaking many ‚Äúpen-and-paper‚Äù ciphers. It doesn‚Äôt guess the key. It measures the shape of the text: which letters and patterns repeat, and how closely that resembles real language.

What it looks at

  • Letter counts (A‚ÄìZ): In English, E/T/A/O/I/N are common. Substitution ciphers keep these counts (just relabeled).
  • Digraphs & trigraphs: Common pairs like TH, HE, IN and triples like THE, AND.
  • Index of Coincidence (IoC): A quick ‚Äúhow clumpy are letters?‚Äù signal. Random text is low; natural language is higher.
  • Chi‚Äësquare vs English: A score of how far the letter distribution deviates from expected English.
  • Repeats: Repeated n‚Äëgrams can hint at transposition vs substitution, and can help key‚Äëlength guesses in polyalphabetic ciphers.

What it tells you (fast triage)

  • Substitution vs transposition: transposition tends to preserve overall letter counts (so frequencies can still look ‚ÄúEnglish-ish‚Äù), while substitution changes which letters are common but the distribution shape remains similar.
  • Is it polyalphabetic (e.g. Vigen√®re)? IoC often drops compared to plain English when multiple alphabets are used, and periodic structure can appear.
  • Is it probably not a classic cipher? Very high entropy / lots of non‚Äëletters can suggest encoding (Base64/Hex), compression, or binary data.

How to use it in the lab

  1. Paste text into Encoder / Decoder to strip/normalize if needed (remove spacing, keep letters only, etc.).
  2. Run analysis: check IoC + chi‚Äësquare first, then scan the top bigrams/trigrams.
  3. Use the result to pick the right tool:
    • Looks monoalphabetic: try Caesar/Affine/Substitution.
    • Looks transposition‚Äëish: try Railfence/Columnar/Permutation.
    • Looks polyalphabetic: try Vigen√®re and test likely key lengths.

Frequency analysis won‚Äôt magically ‚Äúsolve‚Äù everything, but it‚Äôs a powerful filter: it narrows your search space so you don‚Äôt waste time on the wrong family of ciphers.
